5 Levels Of Summarization: Novice to Expert¶

Summarization is a fundamental building block of many LLM tasks. You'll frequently run into use cases where you would like to distill a large body of text into a succinct set of points.

Depending on the length of the text you'd like to summarize, you have different summarization methods to choose from.

We're going to run through 5 methods for summarization that start with Novice and end up expert. These aren't the only options, feel free to make up your own. If you find another one you like please share it with the community.

5 Levels Of Summarization:

  1. Summarize a couple sentences - Basic Prompt
  2. Summarize a couple paragraphs - Prompt Templates
  3. Summarize a couple pages - Map Reduce
  4. Summarize an entire book - Best Representation Vectors
  5. Summarize an unknown amount of text - Agents

Summarize Long Text¶

If you have multiple pages you'd like to summarize, you'll likely run into a token limit.

The chain type "Map Reduce" is a method that helps with this. You first generate a summary of smaller chunks (that fit within the token limit) and then you get a summary of the summaries.

That's too many tokens, let's split our text up into chunks so they fit into the prompt limit. Make a chunk size of 10,000 characters.

You can think of tokens as pieces of words used for natural language processing. For English text, 1 token is approximately 4 characters or 0.75 words. As a point of reference, the collected works of Shakespeare are about 900,000 words or 1.2M tokens.

This means the number of tokens we should expect is 10,000 / 4 = ~2,500 token chunks. But this will vary, each body of text/code will be different

text_splitter = RecursiveCharacterTextSplitter(separators=["\n\n", "\n"], chunk_size=10000, chunk_overlap=500)

docs = text_splitter.create_documents([essay])

Chain Type:

  1. stuff: Pass multiple documents to the LLM at the same time to process. If there are too many documents, it will exceed the token limit. Normally won't use it for summarization chain.

  2. map_reduce: This method will summarize each document separately and then summarize summaries for each document to create the final summary.

  1. refine: This method will summarize first document first, and then use summary of the first document to summarize the second document and so on. The advantage of this method is passing previous document's summary when summarize the current document add context to the summarization task, enhance the consistancy of the summary.
  1. map_rerank: This method is usually used on Q&A chain not summarization chain. Given a question, it will calculate how likely a document can answer this question for every document and find the document with the highest probability. Then pad this document with question to form a prompt (question+document), send it to LLM, and LLM returns an answer.
